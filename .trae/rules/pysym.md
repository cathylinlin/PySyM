# PySyM 项目开发规范

## 1. 项目结构规范

```
PySyM/
├── pyproject.toml
├── README.md
├── LICENSE
├── src/
│   └── PySyM/
│       ├── __init__.py
│       ├── core/             # 核心数学模块
│       ├── abstract_phys/    # 物理抽象模块
│       ├── phys/             # 物理应用模块（根据需求随着调整结构）
│       ├── visual/           # 可视化模块
│       └── tools/            # 工具模块
└── tests/                    # 测试目录

```

## 2. 技术栈限制

### 2.1 核心依赖
- Python 3.8+
- NumPy: 矩阵运算和数值计算
- Scipy: 科学计算库
- SymPy: 符号计算

### 2.2 限制
- 保持核心模块纯Python实现
- 非必要不使用其他第三方库。

## 3. 命名规则

### 3.1 模块命名
- 使用小写字母和下划线

### 3.2 类命名
- 使用大驼峰命名法
- 抽象基类以`Abstract`或`ABC`开头

### 3.3 函数和方法命名
- 使用小写字母和下划线
- 私有方法以单下划线开头

### 3.4 常量命名
- 全大写字母和下划线

### 3.5 变量命名
- 使用小写字母和下划线
- 物理量使用标准符号(如`psi`, `phi`)

## 5. 物理应用开发指南

### 5.1 物理模块结构
- 每个物理领域独立模块
- 核心数学功能在`core`模块
- 物理应用复用核心功能

### 5.2 物理过程建模
- 明确物理假设
- 记录近似条件
- 提供验证方法

这些规范将确保项目的一致性、可维护性和可扩展性，同时支持对称性理论在物理学中的应用开发。
